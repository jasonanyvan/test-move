Motion

Use motion to reduce cognitive load and add moments of delight

Always use prefers-reduced-motion to give users an escape hatch from animation

Avoid animating any property except for opacity, colour and transform

Create quick and smooth transitions by using the cubic easing with faster durations

If a user's action triggers a negative response, don't use any motion. Being sincere and direct is the best approach

Real world objects usually accelerate and decelerate as they move. We try and mimic this through easing. Easing is defined as the way an animation changes speed over the course of the action. We use three types of easing throughout our digital ecosystem

In

Ease-in animations work best for fading and animating objects into view - this is our most common used easing method

cubic-bezier(0.6, 0, 1, 1)

Out

Ease-out animations are best suited for objects exiting the screen or fading out of view

cubic-bezier(0, 0, 0.6, 1)

Hybrid

Ease-in-out easing is best for depicting movement that happens between two points on screen. This type of easing draws attention and should only be used for larger movements

cubic-bezier(0.4, 0, 0, 1)
