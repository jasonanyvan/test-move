Grids

There are 3 primary containers that make up layouts within the AnyVan ecosystem - [1] navigation, [2] app bars and [3] content

App bars are very much a container that lives within the adminstrative side of the AnyVan applications and don't appears on more client facing / marketing efforts. In lieu of app bars, we can use secondary navigation elements to achieve the interactivity of the app bar on our marketing efforts

Extra large Large

Default layout

A standard 12 column grid. The maximum width of the 12 columns should be 78% of the screen width and each column should have a gutter of 24px

Narrow layout

In this layout, the navigation container should maintain a consistent width of 80px. The ideal navigation compontent for this layout is the navigation rail

Expanded layout

In this layout, the navigation container should maintain a consistent with of width of 300px - columns should be anchored left with an offset of 360px to account for the navigation. This ideal navigation compontent for this layout is the navigation drawer

Medium Small

Default layout

A standard 12 column grid. The maximum width of the grid columns should be determined by outer margins of 24px, left and right, and each column should have a gutter of 12px

Extra small

Default layout

A standard 4 column grid. The maximum width of the grid columns should be determined by outer margins of 18px, left and right, and each column should have a gutter of 15px

Narrow layout

In this layout, the navigation container should maintain a consistent height of 57px. The ideal navigation compontent for this layout is the primary navigation bar. As an alternative, on alpha pages, a bottom app bar can also we used for quick actions and navigation
