Navigation schemes

Alpha

Alpha pages or states are defined as top level pages — the first pages in a hierarchy, that usually have child pages. The navigation scheme, the hamburger menu, seen on these pages in the primary navigation bar below, should be the default for alpha pages. This only applies to top level pages on desktop that don't use navigation rails or expanded navigation drawers

Beta

Moving a level deeper, beta pages are pages that follow on from alpha pages, one down in the navigation hierarchy, and can exclude the hamburger menu, seen in the app bars below. Where possible try and keep all applications and executions to a maximum depth of 2 — this helps users navigation with greater ease and allows them to find what they need without digger through too many layers

Gamma

For more detailed schemes, consider using breadcrumbs to highlight a users path through the app. Alternatively, try nested side navigation, within a beta page, to hop between content that can be combined into sections as opposed to creating new pages. This scheme should be seen as edge case
